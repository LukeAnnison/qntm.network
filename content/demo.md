# qntm.network markdown demo

This page is rendered from **markdown** into HTML *in your browser*, by a single TypeScript
renderer. Press **Edit** to see the source that produced it — and change it.

## Try it

Switch to edit mode and type. The document re-renders as you go, because the renderer is a pure
function and the documents are small, so it can simply run again on every keystroke.

Nothing you type is saved. There is no `localStorage`, no draft, no sync — refresh the page and
the original comes back. That is a deliberate property of this version rather than a missing
feature, and it is enforced: a check fails the build if a persistence API ever appears in the
app's source, so saving cannot arrive by accident.

## What it demonstrates

- headings (three levels here)
- lists, like this one
- **bold** and *italic*
- inline `code`
- [links](https://qntm.network)

### A fenced code block

```ts
import { MarkdownRenderer } from "./render/renderer.js";

new MarkdownRenderer().toHtml("# hello"); // pure: markdown -> html
```

### A table

| piece | what it does | knows about the DOM |
|-------|--------------|:-------------------:|
| `MarkdownRenderer` | markdown -> HTML, pure | no |
| `EditorSession` | holds the mode and the text | no |
| `main.mount` | paints the page, wires the toggle | yes |

> Keeping the DOM in one place is what lets the other two be reasoned about — and tested —
> without a browser.

### What was here before

Until 2026-07-23 this page was generated by a build-time Python renderer, and this file said so.
That renderer retired the day editing arrived: a live editor cannot re-run a Python build on
every keystroke, and keeping both would have left two implementations of one contract to drift
apart. Retiring it is why there is exactly one renderer to point at above.
